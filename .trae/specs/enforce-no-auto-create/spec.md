# 强化文件创建限制 Spec

## Why
当前系统提示词中已有禁止自动创建测试文件和 README.md 的说明，但需要更加明确和强调这一规则，确保 AI Agent 在任何情况下都不会未经用户明确要求就创建额外文件。

## What Changes
- 在系统提示词中更加突出和强调"禁止自动创建文件"的规则
- 明确说明只有在用户**明确要求**时才创建测试文件、README.md 等文档
- 将此规则放在更显眼的位置，使用更强烈的措辞

## Impact
- 受影响的功能：Agent 模式下的文件创建行为
- 影响范围：所有通过 AI Agent 创建文件的场景
- 用户体验：避免用户被自动创建的不必要文件困扰

## ADDED Requirements

### Requirement: 文件创建限制强化
系统提示词必须明确且突出地说明：
- 除非用户**明确要求**，否则禁止创建任何测试文件
- 除非用户**明确要求**，否则禁止创建 README.md、CHANGELOG.md 等文档文件
- 只有在用户明确说出"创建测试文件"、"编写测试"、"添加 README"等指令时，才允许创建对应文件

#### Scenario: 用户要求创建组件
- **WHEN** 用户说"创建一个用户组件"或"实现一个登录功能"
- **THEN** AI 只创建实现该功能所必需的核心文件
- **THEN** AI 不得自动创建测试文件、文档文件、配置文件等额外文件

#### Scenario: 用户明确要求测试
- **WHEN** 用户说"创建一个用户组件，并编写测试"或"实现功能后添加测试用例"
- **THEN** AI 可以创建测试文件
- **THEN** 测试文件应该与主文件在同一请求中创建

### Requirement: 提示词位置优化
- 将文件创建限制规则放在系统提示词的靠前位置
- 使用更醒目的标记（如【重要】、【必须遵守】等）
- 在 Agent 模式系统提示词中作为核心原则强调

## MODIFIED Requirements

### Requirement: 系统提示词 - 文件创建原则
原规则：
```
【文件创建原则 - 非常重要】
- **只创建用户明确要求的文件**，不要自动创建任何额外的文件
- 禁止自动创建：测试文件（*.test.ts, *.spec.ts, test/*.js 等）、README.md、CHANGELOG.md、.gitignore、许可证文件等
```

修改为：
```
【文件创建原则 - 必须严格遵守】
- **绝对禁止**创建用户未明确要求的任何文件
- **除非用户明确要求**，否则禁止创建以下类型的文件：
  - 测试文件（*.test.ts, *.spec.ts, test/*.js, tests/*.py 等）
  - 文档文件（README.md, CHANGELOG.md, docs/*.md 等）
  - 配置文件（.gitignore, .eslintrc, tsconfig.json 等）
  - 许可证文件（LICENSE, LICENSE.txt 等）
- 如果用户说"创建一个组件"或"实现一个功能"，**只创建**实现该功能的核心代码文件
- 只有当用户明确说出"创建测试"、"编写测试用例"、"添加 README"等指令时，才允许创建对应文件

```
